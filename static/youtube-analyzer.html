1단계 · OAuth 클라이언트 ID 입력

Google Cloud Console에서 발급한 OAuth 클라이언트 ID 하나만 있으면 됩니다. 입력값은 브라우저에서만 사용되며 외부로 전송되지 않습니다.

📋 클라이언트 ID 발급 절차
  • 1 Google Cloud Console 접속 → 상단 프로젝트 선택 메뉴에서 새 프로젝트 만들기 (이름 예: YouTube-Analyzer). 기존 프로젝트가 있으면 재사용 가능.
  • 2 왼쪽 메뉴 → API 및 서비스 → 라이브러리 → 검색창에 YouTube Data API v3 입력 → 결과 클릭 → 사용 설정 버튼.
  • 3 API 및 서비스 → OAuth 동의 화면
    ① User Type: 외부 선택 → 만들기
    ② 앱 이름·사용자 지원 이메일·개발자 연락처에 본인 Gmail 입력
    ③ 범위 추가 또는 삭제 → youtube.readonly 검색 후 체크 → 업데이트
    ④ 테스트 사용자에 본인 Gmail 추가 (최대 100명까지 등록 가능)
  • 4 API 및 서비스 → 사용자 인증 정보 → 상단 + 사용자 인증 정보 만들기 → OAuth 클라이언트 ID
    ① 애플리케이션 유형: 웹 애플리케이션
    ② 이름: 자유 (예: Web Client)
    ③ 승인된 JavaScript 출처에 아래 두 개 모두 추가 http://127.0.0.1:5000 http://localhost:5000 ④ 만들기 → 표시된 클라이언트 ID(숫자-문자열.apps.googleusercontent.com) 복사 → 위 입력칸에 붙여넣기

⚠️ 클라이언트 시크릿은 사용하지 않습니다. ID만 복사하세요.
⚠️ "이 앱은 확인되지 않았습니다" 경고가 떠도, 테스트 사용자로 등록된 본인 계정이면 고급 → 안전하지 않음으로 이동으로 진행 가능합니다.

2단계 · Google 계정 연결
아직 연결되지 않았습니다.

클릭하면 Google 로그인 팝업이 열립니다.
YouTube 구독 데이터 읽기 권한(youtube.readonly)을 허용해주세요.

3단계 · 분석 시작
📁 (선택) Google Takeout 파일 업로드
업로드하면 정확한 댓글 수와 채널별 마지막 시청일을 점수에 반영합니다.
takeout.google.com → "YouTube 및 YouTube Music" → comments · history 선택 → 내보내기.
압축파일 안의 comments.csv / watch-history.json 을 한 개씩 또는 둘 다 업로드 (재선택으로 추가 가능).
업로드하지 않으면 댓글은 API 근사값, 시청 기록은 미반영.

구독 채널 수에 따라 10초~1분 정도 소요될 수 있습니다.
