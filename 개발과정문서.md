## 1. 프로젝트 개요

Google OAuth로 본인 YouTube 계정에 연결한 뒤, 구독 채널을 **댓글·좋아요·구독 기간·시청 기록**으로
종합해 **관심도 점수(0~100)** 로 환산하고 4그룹(🔥/😊/👋/🌱)으로 자동 분류해 주는 단일 페이지 웹 앱입니다.
여기에 **Upstage Solar LLM** 기반 채널 추천·페르소나 분석·구독 정리, 그리고 **카테고리 분류·코사인 유사도
추천·취향 커뮤니티(Union-Find)·취향 지도(MST)** 까지 결합한 취향 분석 도구입니다.

- **Frontend**: Vanilla JS + HTML/CSS (단일 페이지), force-directed SVG 그래프 시각화
- **Backend**: Python Flask (`/api/analyze`, LLM 프록시, 그래프 분석 등)
- **LLM**: Upstage Solar
- **인증**: Google Identity Services(GIS) 토큰 클라이언트, YouTube Data API v3

## 3. 주차별 개발 내역

### Week 09 — 기획 & 셋업
- 주제 브레인스토밍 후 "YouTube 구독 채널 관심도 분석" 확정.
- GitHub Public 레포 생성, 초기 README 작성, Solar/NIM API 신청.

### Week 10 — 요구사항 & 화면 설계
- 핵심 기능 3가지 정의: ① 관심도 점수화·자동 그룹핑, ② Solar 기반 채널 추천, ③ 구독 정리·페르소나 분석.
- 데이터 소스 조사: YouTube Data API v3 + Google Takeout(comments.csv, watch-history.json) 하이브리드 전략 결정.
- 점수 산정 방식 초안(댓글·좋아요·구독기간 가중 합산) 설계.

### Week 11 — 프로토타입
- 단일 `youtube-analyzer.html`로 OAuth 로그인 → 구독 목록 호출 → 화면 렌더링까지 동작하는 프로토타입 완성.
- API 호출 흐름과 UI 레이아웃 검증.

### Week 12 — MVP (백엔드 + 알고리즘)
- 프로토타입을 **Flask 서버(`app.py`) + 정적 프론트(`static/app.js`, `style.css`)** 구조로 분리.
- **핵심 알고리즘 구현**:
  - 활동 합산 + **휴면 채널 0점 단락(short-circuit)**
  - **Min-Max 정규화** → 점수 0~100 스케일링
  - **Binary Search** 기반 임계값 4그룹(🔥/😊/👋/🌱) 분류
  - **Heap Sort** 내림차순 정렬
  - 입력 의존 키 **LRU 캐시**로 재분석 비용 절감
- Takeout 업로드(시청 기록) 시 **Recency Decay**(최근 시청 가중) 반영.
- **페르소나 분석·구독 정리 기능 개발 착수** (커밋은 6/2 `58cae7a`에 병합 기록).

### Week 13 — LLM 통합 & 추천·그래프 개발
- **Upstage Solar 연동**: Flask 프록시 경유 채널 추천, OAuth 설정 가이드 문서화 (`a411fca`, 05/26).
- `.env.example`, `.gitignore` 정비로 비밀키·개인 데이터 추적 제외.
- LLM 분석 도구 확장: **페르소나 분석·구독 정리 기능 완성** (커밋은 6/2 `58cae7a`).
- 채널 추천을 **영상 우선 발굴(video-first discovery)** 구조로 전면 개편 (커밋은 6/2 `16541df`).
- **시청기록 기반 채널 추천**과 **카테고리 다양성 보정** 로직 개발 (커밋은 6/2 `41495ed`).
- **카테고리 분류 + 코사인 유사도 추천 개발 착수** (`category_list.yaml` 사전 구축 시작).
- **`feat/taste-graph` 별도 브랜치**에서 **Union-Find 취향 커뮤니티·Kruskal MST 취향 지도** 개발 착수.

### Week 14 — 그래프 완성·영어 발표·마무리

## 5. 적용 알고리즘 정리 (요구사항 5번 대응)

| 알고리즘 개념 | 적용 위치 | 목적 |
|---------------|-----------|------|
| 정렬 (Heap Sort + 안정 정렬 2차키) | 관심도 점수 내림차순 정렬, 동점 안정화 | 결과 랭킹 |
| 탐색 (Binary Search) | 임계값 기반 4그룹 분류 | O(log n) 구간 판정 |
| 해싱 / 캐시 (LRU) | `/api/analyze` 입력 키 캐시 | 재분석 비용 절감 |
| 정규화 (Min-Max) | 점수 0~100 스케일링 | 채널 간 비교 가능 |
| 가중 감쇠 (Recency Decay, `exp(-λ·d)`) | 시청 기록 기반 가중 | 최근 관심 반영 |
| 문자열 매칭 (카테고리 분류) | 키워드 규칙 기반 채널 카테고리화 | 취향 벡터 생성 |
| 벡터 유사도 (코사인 유사도) | 카테고리 벡터 기반 채널 추천 | 취향 유사 채널 발굴 |
| **그래프 — Union-Find (경로압축·랭크)** | 유사도 그래프 연결요소 군집화 | **취향 커뮤니티** 도출 |
| **그래프 — Kruskal 최대 신장 포레스트(MST)** | 강한 유사도 우선 채널 연결 | **취향 지도** 트리 구성 |
| 시각화 (Force-directed 레이아웃) | SVG 노드 배치·렌더 | 취향 지도 시각화 |

## 6. 회고 (Retrospective)

- 프로토타입(Week 11) → MVP(Week 12) → LLM 확장(Week 13) → 취향 그래프 고도화(Week 14)의 점진적 개발.

- API 한계를 Takeout 하이브리드로 우회하는 등 실용적 문제 해결.

- 브랜치 분리를 일부 기능에만 적용 → 모든 기능에 기능 단위 브랜치 + PR 리뷰 표준화 필요.
- 자동화된 테스트 부재 → 회귀 검증을 수동에 의존.
